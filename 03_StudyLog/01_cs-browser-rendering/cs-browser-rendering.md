# 브라우저 렌더링 과정에 대해

각각의 용어에 대한 개념과 그 과정에 대해 알아보자.

### 1. 브라우저(Browser)

인터넷에서 웹 페이지, 이미지, 동영상 등 다양한 콘텐츠를 검색하고 표시하는 소프트웨어 프로그램이다. 우리가 입력한 URL을 해석하고 서버와 통신해 최종적으로 콘텐츠를 화면에 띄워주는 도구이다.

- 대표적인 브라우저
  인터넷 익스플로러, 파이어폭스(Firefox), 사파리(Safari), 크롬(Chrome), 오페라(Opera) 등

### 2. 렌더링(Rendering)

렌더링은 HTML, CSS, JavaScript 같은 문서를 해석해 실제 화면에 출력하는 과정이다. 웹 브라우저가 HTML, CSS, JavaScript 등의 소스 코드를 받아 해석하고 최종적으로 사용자가 볼 수 있는 화면으로 바꾸는 것이다.

### 3. 브라우저 렌더링

브라우저 렌더링은 사용자가 URL을 입력한 순간부터 화면에 결과가 나타나기까지의 과정을 말한다.

**1) 파싱 (Parsing)**

- HTML 파싱
  브라우저는 서버에서 받은 HTML 문서를 위에서부터 한 줄씩 해석한다. 태그(`<html>`, `<body>`, `<div>` 등)를 만나면 이를 객체 형태로 변환하여 DOM(Document Object Model) 트리를 생성한다.
- CSS 파싱
  `<link>`나 `<style>`로 불러온 CSS를 해석해 CSSOM(CSS Object Model) 트리를 만든다. CSS의 상속, 우선순위, 캐스케이딩 규칙을 모두 고려해 각 요소에 어떤 스타일이 적용될지 계산할 준비를 한다.

**2) 렌더 트리(Render Tree) 생성**

DOM + CSSOM을 결합해 실제 화면에 표시할 요소만 모아 렌더 트리(Render Tree)를 만든다.

- `display: none;` 같이 화면에 나타나지 않는 요소는 렌더 트리에 포함되지 않는다.
- `visibility: hidden;` 처럼 공간은 차지하지만 보이지 않는 경우는 여전히 렌더 트리에 남아있다.

**3) 레이아웃 (Layout, Reflow)**

렌더 트리를 기반으로 각 요소의 위치와 크기를 계산한다. 브라우저는 뷰포트 크기, 박스 모델(Box Model), Flexbox, Grid 같은 레이아웃 규칙을 적용한다.

**4) 페인팅 (Painting)**

레이아웃 단계에서 계산된 정보를 바탕으로 픽셀 단위로 스타일을 그리는 것. 색상, 텍스트, 테두리, 그림자, 이미지, 배경 등을 그려낸다.

**5) 합성 (Compositing)**

브라우저는 화면을 여러 레이어로 나누고 GPU를 이용해 이들을 빠르게 합성한다. 이 과정에서 스크롤, 애니메이션, CSS transform/opacity 같은 효과가 GPU 가속을 활용해 부드럽게 처리된다.

### 4. 렌더링 최적화 관점에서 중요한 점

- **리플로우(Reflow) 최소화**
  자바스크립트로 돔을 변경하거나 CSS에서 위치 값에 영향을 주는 속성들을 변경하는 등 레이아웃에 영향을 미치게 되면 **레이아웃 → 페인트 → 합성** 과정이 재실행되는 리플로우가 발생한다.
  - 리플로우를 발생시키는 대표적인 속성 : `position, width, height, margin, padding, border, border-width, font-size, font-weight, line-height, text-align, overflow`
- **리페인트(Repaint) 최소화**
  레이아웃에 영향을 미치지 않는 속성들을 변경할 때에는 **페인트 → 합성** 과정이 재실행되는 리페인트가 발생한다.
  - 리페인트를 발생시키는 대표적인 속성 : `background, color, text-decoration, border-style, border-radius`

> 리플로우는 윈도우 리사이즈, DOM 구조 변경, CSS 속성 변경 등으로 이미 계산한 레이아웃을 다시 계산해야 하는 상황

## 끝으로

브라우저 렌더링 과정은 눈에 보이지 않는 순간에 이루어지지만 프론트엔드 개발자가 반드시 이해해야 하는 기본기다. 렌더링 파이프라인을 알고 있으면 코드가 어떤 영향을 미치는지 예상할 수 있고 성능 최적화에도 도움이 된다.
