문제: "Java" vs "java" vs "JAVA " - 모두 다른 결과

해결: 정규화 파이프라인


private String joinKeywords(List<String> keywords) {
    return keywords.stream()
        .map(String::trim)          // 공백 제거
        .map(String::toLowerCase)   // 소문자 변환
        .distinct()                 // 중복 제거
        .collect(Collectors.joining(","));
}

결과: 저장 시점에 정규화 → 검색 품질 100% 일치

3. 추천 시스템 BE

알고리즘: Jaccard Similarity - 태그 기반 유사도 계산
J(A,B) = |A ∩ B| / |A ∪ B|

교집합 크기 / 합집합 크기

구현 코드


public RecommendationResponse getRecommendations(Long shortsId) {
    Set<String> baseKeywords = extractKeywords(baseShorts);

    return candidates.stream()
        .map(c -> {
            Set<String> target = extractKeywords(c);
            double score = calculateJaccard(baseKeywords, target);
            return new SimilarityResult(c.getId(), score);
        })
        .filter(r -> r.similarity > 0)
        .sorted(desc())
        .collect(toList());
}

Infrastructure

안정적인 서비스 운영 환경

인프라 아키텍처

  • Compute: AWS EC2 + Docker (컨테이너 기반 배포)
  • Processing: AWS Lambda (영상 처리 별도 분리)
  • Storage: AWS S3 (영상/썸네일 원본 저장)
  • Database: MySQL (RDB) + Redis (Cache/Session)

Serverless 썸네일 처리

Why Lambda?
EC2에서 FFmpeg 실행 시 CPU/메모리 점유율 급증
→ 업로드 이벤트 트리거로 분리하여 서버 부하 0% 달성

처리 프로세스

Flow

  1. User가 S3에 영상 업로드 (Presigned URL)
  2. S3 Put Event가 Lambda 트리거
  3. Lambda가 첫 프레임 추출 → 썸네일 S3 저장
  4. 서버는 약속된 경로로 썸네일 URL 접근

Benefit

  • 메인 서버 리소스 보호
  • 병렬 업로드 시 확장성 보장
  • 실패 시 재시도 로직 분리 용이
